# 测试文档

后端集成测试为主，**贴着真实依赖跑**（真 PostgreSQL + 真 Redis），外部不可控部分用忠实替身（ldap3 内存目录、moto 模拟 S3、`hermes` 替身二进制 / mock ACP 代理）。

---

## 1. 测试栈

| 工具 | 用途 |
|---|---|
| pytest · pytest-asyncio | 异步测试（`asyncio_mode=auto`） |
| httpx `ASGITransport` | 进程内驱动 FastAPI（单事件循环，避免 TestClient 多循环坑） |
| moto `mock_aws` | 模拟 S3 验证 MinIO 存储路径 |
| ldap3 `MOCK_SYNC` | 内存 LDAP 目录验证域登录 |
| websockets | 真 WebSocket 客户端（圆桌实测） |

---

## 2. 如何运行

需要可达的 PostgreSQL + Redis，并已迁移 + seed：
```bash
cd backend
export DATABASE_URL=postgresql+asyncpg://hermes:hermes@localhost:5432/hermes
export REDIS_URL=redis://localhost:6379/0
export SECRET_KEY=test-secret
alembic upgrade head && python -m app.seed
pip install -e ".[dev]"          # pytest, pytest-asyncio, moto, ruff
pytest -q                        # 全部
pytest tests/test_roundtable.py -v
```
依赖不可达时，相关用例 `skip`（不失败），便于无依赖环境下也能收集。

### conftest 关键
`tests/conftest.py` 有 autouse fixture，**每个用例前后重置全局 engine/redis 连接池**——因为 pytest-asyncio 每用例一个事件循环，而模块级全局池绑定首次使用的循环。生产是单循环，无此问题。

## 4. 替身与隔离策略

- **ACP / agent**：默认用 `agent_runner/mock_agent.py`（真 ACP stdio 服务端，支持 persona）。真实路径用 `tests/test_real_cli_integration.py` 里临时写一个名为 `hermes` 的可执行文件放上 PATH，证明 `acp_cli` 分支（而非 mock 兜底）。
- **LDAP**：`identity_service.LDAP._factory` 注入一个 `MOCK_SYNC` 连接工厂，内存里加一个带 `userPassword` 的用户条目；bind 校验密码。
- **S3/MinIO**：`@mock_aws` + `minio_endpoint=""`（走默认端点让 moto 拦截）。
- **跨次运行的数据累积**：共享 DB 不清库，测试用唯一标识（uuid/时间戳邮箱、唯一关键词）或显式清理（如 identity 用例结束禁用 provider、删映射）保证幂等。

## 6. 端到端验收

仓库根可用「全栈验收」方式（原生进程，等价 compose）逐项打勾 P1–P5：登录 → 单聊 SSE 流式 → 工作区预览 → 团队/权限 → 圆桌 WebSocket → 后台/限流/审计/历史 → LDAP/指标。脚本思路见 [运维文档](运维文档.md) 与提交记录中的验收说明。
