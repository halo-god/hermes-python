# 数据库设计

PostgreSQL 16。模型在 `backend/app/db/models/`，迁移在 `backend/alembic/versions/`（0001–0006，线性串联）。
主键多为 UUID（`gen_random_uuid()`，pgcrypto）；时间戳 `created_at/updated_at`（带时区）。

---

## 3. JSONB 字段一览（为何用 JSONB）

| 表.字段 | 结构 | 原因 |
|---|---|---|
| messages.content | `{text}` 或 `{replies[],merged}` | 消息形态多样（单/圆桌/工具调用） |
| teams.policy | `{perm_id:{role_key:bool}}` | 权限矩阵随权限点演进 |
| system_settings.data | 品牌/模型网关嵌套 | 租户配置灵活 |
| agents.command | `["hermes","acp"]` | spawn argv |
| projects.sections / pinned_agents | 数组 | 轻量列表 |
| identity_providers.config | 连接参数 | 各提供商字段不同 |

> 迁移里写 JSONB 字面量务必 `CAST(:d AS jsonb)` + `json.dumps(...)`，避免双重编码（见 `0005_admin.py` 经验）。

## 4. 索引

- `users(email)` unique、`users(handle)` unique、`users(source,external_id)` unique
- `conversations(owner_id)`、`messages(conversation_id)`、`workspace_files(conversation_id)`
- `projects(team_id)`、`project_tasks(project_id)`、`dept_team_mappings(provider_id)`
- `audit_log(ts)`、`audit_log(action)`
- teams.handle unique

后续可按需加：会话标题全文检索（`pg_trgm`/`tsvector`）、知识库向量（`pgvector`）。
