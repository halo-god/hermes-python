# 安全文档

系统的安全模型、控制点与生产加固清单。最大攻击面是 **agent 子进程会执行代码 / 读写文件 / 联网**——务必沙箱。

---

## 1. 认证

- **密码**：argon2id（argon2-cffi）哈希，登录时透明升级参数（`needs_rehash`）。SSO-only 用户无本地密码（`password_hash=NULL`）。
- **令牌**：JWT（HS256）。access 短时效（默认 15 min）+ refresh（默认 7 天）。
  - **refresh 轮换**：每次刷新签发新 refresh，旧的写入 Redis 黑名单（`jwt:blacklist:{jti}`，TTL=自然过期）；重用旧 refresh → 401。
  - **登出 / 踢设备**：把 refresh jti 加黑名单。
- **多副本一致性**：所有 `api` 实例必须用**同一 `SECRET_KEY`**，否则跨实例令牌校验失败。

---

## 2. 授权（双层）

1. **平台 RBAC**（`app/core/rbac.py`）：`super_admin > admin > team_admin > member > viewer`。后台接口用 `require_admin()` 守卫。
2. **团队内容权限矩阵**（`app/core/governance.py`）：`perm × role` 策略存 `teams.policy`，`team_service.require_permission(...)` 守卫每个团队写操作。owner 列恒为真且锁定；owner/admin 可改矩阵。

> 服务层强制按 `owner_id`/`team_id` 作用域查询，杜绝越权读取他人数据（多租户隔离）。

---

## 3. Agent 沙箱（生产红线）

agent 子进程 = 最大攻击面。分层防护（`agent_runner/sandbox.py`，配置见[运维文档 §4](运维文档.md)）：

1. **POSIX rlimits**（`SANDBOX_ENABLED=true`）：CPU、进程数、文件大小（可选地址空间）。
2. **隔离包装**（`SANDBOX_CMD`）：bubblewrap / firejail / gVisor `runsc` 前缀，`{cwd}` 模板可绑定工作目录，做文件系统/网络隔离。
3. **运行时隔离**：K8s 用 **gVisor RuntimeClass** 跑 agent-runner，配 **NetworkPolicy** 限制出口（防数据外泄 / SSRF）。
4. **每会话独立工作目录**（`WORKSPACE_ROOT/{conversation_id}`），用户/团队间文件系统不互通。
5. **路径穿越防护**：`fs/write_text_file` 的 path 落工作目录内（按 basename 处理）。

> 默认 `SANDBOX_ENABLED=false` 仅为开发/离线便利；**生产必须开启并配隔离运行时**。

---

## 4. 凭证管理

- **平台密钥**：`SECRET_KEY`、DB/Redis/MinIO 口令走环境/密钥管理（KMS/Secret），不入库、不入日志。
- **Agent 凭证**：真实 `hermes` CLI 的模型 Key（`HERMES_API_KEY`/`ANTHROPIC_API_KEY`/…）以租户维度注入 runner 进程环境，**不经 ACP、不落日志**（见[真机联调](真机联调.md)）。
- **身份提供商配置**：`identity_providers.config`（含 LDAP bind 口令等）存库；生产应对敏感字段加密 / 仅管理员可读，并在返回时按需脱敏。

---

## 5. 限流与配额

- 每用户每分钟发送闸门（Redis 固定窗口，`rl:msg:{user}`），超限 429。
- 限流阈值运行时可调（后台「系统设置」→ 写 Redis `cfg:rate_limit_per_min`），无需重启。
- 月度用量计数（`usage:msg:{user}:{YYYYMM}`）用于配额展示 / 软限制。
- 建议在网关再加 IP 级限流防爆破。

---

## 6. 传输与边界安全

- **TLS**：网关/`web` 前置终止 HTTPS；WebSocket 用 `wss`。
- **CORS**：`CORS_ORIGINS` 收敛到真实前端域。
- **SSE/WS 令牌**：因 EventSource 不能设 header，令牌经查询参数传递——务必全程 HTTPS，避免日志记录完整 URL（或改用短时一次性 stream-token，作为后续增强）。
- **反代**：SSE 关闭缓冲（已带 `X-Accel-Buffering: no`），WS 放行 Upgrade 头。
- 内部依赖（PG/Redis/MinIO）只走内网，不暴露公网。

---

## 7. 审计

安全相关动作写不可变 `audit_log`：登录成功/失败（含 IP、原因）、管理员建/改用户、改系统设置、改身份提供商。`GET /admin/audit` 可按 action/result 过滤。建议定期外送 SIEM。

---

## 8. 依赖与供应链

- 后端依赖钉版本（`pyproject.toml`），前端有 `package-lock.json`。
- 建议 CI 接 `pip-audit` / `npm audit` / 镜像扫描。
- agent-runner 镜像装真实 CLI 时，安装脚本来自官方源（见 Dockerfile），生产应锁定版本 / 校验。

---

## 9. 生产安全清单

- [ ] `SECRET_KEY` 强随机且多副本一致；`DEBUG=false`、`ENVIRONMENT=production`
- [ ] 改默认管理员密码；强密码策略
- [ ] `CORS_ORIGINS` 收敛；全站 HTTPS/WSS
- [ ] `SANDBOX_ENABLED=true` + gVisor/bwrap + NetworkPolicy 限出口
- [ ] `ACP_ALLOW_MOCK_FALLBACK=false`（生产不容忍静默 mock）
- [ ] DB/Redis/MinIO 内网 + 强口令；敏感配置加密
- [ ] agent/模型凭证经 KMS 注入，不落日志
- [ ] 限流（应用 + 网关）；审计外送
- [ ] 依赖/镜像扫描；定期备份与恢复演练
- [ ] 最小权限：服务账号、DB 账号按需授权
