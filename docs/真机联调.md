# 真机联调：接入真实 Hermes Agent CLI

把 agent-runner 从内置 **mock ACP 代理**切换到 **NousResearch 真实 `hermes` CLI**（`hermes acp`，stdio JSON-RPC）。

> 默认（开发态）：未安装 `hermes` 时，runner 自动回退到内置 mock 代理（`agent_runner/mock_agent.py`），全栈离线可跑。
> 真机态：在 agent-runner 镜像里装真实 CLI，并关闭 mock 兜底。

---

## 1. 工作原理

```
API ──Redis Stream──> agent-runner ──spawn──> `hermes acp`  (子进程, stdio JSON-RPC)
                          │  discovery: shutil.which("hermes") → [path, "acp"]
                          │  ACPClient: initialize → (authenticate?) → session/new → session/prompt
                          └──Redis Pub/Sub──> SSE / WebSocket ──> 浏览器
```

- **探测**：runner 启动时扫描 `PATH` 找 `HERMES_BIN`（默认 `hermes`），探测 `hermes --version`，注册为 `kind=acp_cli`，启动命令 `[<path>, "acp"]`。
- **会话**：每个会话一个长驻 `hermes acp` 子进程，保留上下文；产物经 `fs/write_text_file` 落工作区。
- **凭证**：真实 CLI 自行鉴权（`hermes login` 或 API Key 环境变量），在主机/容器层注入，**不经 ACP**。

---

## 2. 启用步骤（Docker Compose）

```bash
cp .env.example .env
# 编辑 .env：
#   INSTALL_HERMES=true            # 构建时装真实 CLI（需要外网）
#   ACP_ALLOW_MOCK_FALLBACK=false  # 找不到 hermes 时报错，而非用 mock
#   HERMES_API_KEY=...  (或 ANTHROPIC_API_KEY / OPENAI_API_KEY，按你的 hermes 配置)

# 重新构建并启动（agent-runner 镜像会执行官方 install.sh）
docker compose -f docker/compose.yaml --env-file .env build agent-runner
make up
```

`agent-runner.Dockerfile` 的安装片段（已内置，受 `--build-arg INSTALL_HERMES` 控制）：
```dockerfile
ARG INSTALL_HERMES=false
RUN if [ "$INSTALL_HERMES" = "true" ]; then \
      curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash ; \
    fi
ENV PATH="/root/.local/bin:${PATH}"
```

### 鉴权（二选一）
- **环境变量**：把模型 Key 通过 `.env` 传入 agent-runner（compose 已透传 `HERMES_API_KEY` / `ANTHROPIC_API_KEY` / `OPENAI_API_KEY`）。
- **登录态挂载**：在主机 `hermes login` 后，把 `~/.hermes` 挂进容器：
  ```yaml
  agent-runner:
    volumes:
      - ${HOME}/.hermes:/root/.hermes:ro
  ```

## 3. 验证清单

启动后逐项确认：

1. **探测成功**（runner 日志）：
   ```
   hermes.discovery — Discovered ACP agent: hermes -> /root/.local/bin/hermes ['acp']
   hermes.runner — Registered 1 agent(s): hermes
   ```
   不应再出现 `using bundled mock ACP agent`。

2. **注册表**：`GET /api/v1/agents` 中 `hermes` 的 `kind` 应为 `acp_cli`（不是 `builtin_mock`），`version` 为真实版本号。

3. **真对话**：登录 → 新建会话 → 发送 → SSE 应流式返回真实模型输出，`fs/write_text_file` 产物进入工作区；会话 `acp_session_id` 被记录。

4. **健康**：`GET /metrics` 有 `hermes_messages_total` 增长；runner 日志无 `subprocess closed — stderr: ...` 报错。

## 4. 配置项参考

| 环境变量 | 默认 | 说明 |
|---|---|---|
| `INSTALL_HERMES` | `false` | 构建 agent-runner 时是否装真实 CLI（build arg） |
| `HERMES_BIN` | `hermes` | 探测的二进制名 |
| `HERMES_ACP_ARGS` | `["acp"]` | 进入 ACP 服务的参数（真实 CLI 为 `hermes acp`） |
| `HERMES_ACP_AUTH_METHOD` | 空 | 仅当 agent 通过 ACP `authMethods` 要求鉴权时填写 |
| `ACP_ALLOW_MOCK_FALLBACK` | `true` | 生产置 `false`：缺 CLI 时报错而非 mock |
| `HERMES_API_KEY` / `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` | 空 | 透传给 runner，供真实 CLI 鉴权 |
| `SANDBOX_ENABLED` / `SANDBOX_CMD` | `false` / 空 | agent 子进程沙箱（见 §5） |

---

## 5. 沙箱（强烈建议生产开启）

真实 CLI 会执行代码、读写文件、联网——务必隔离（方案 §12）：
- `SANDBOX_ENABLED=true`：对每个 `hermes acp` 子进程施加 POSIX rlimits（CPU/进程数/文件大小）。
- `SANDBOX_CMD`：用 bubblewrap/firejail/gVisor `runsc` 包装，做文件系统/网络隔离，例如
  `SANDBOX_CMD="bwrap --unshare-net --ro-bind / / --bind {cwd} {cwd}"`（按需调整）。
- K8s：用 **gVisor RuntimeClass** 跑 agent-runner，配合 NetworkPolicy 限制出口。

## 6. 排错

| 现象 | 原因 / 处理 |
|---|---|
| 日志仍 `using bundled mock ACP agent` | `hermes` 不在 PATH。确认 `INSTALL_HERMES=true` 构建成功、`PATH` 含安装目录；容器内 `which hermes`。 |
| `subprocess closed — stderr: ...` | runner 已把子进程 stderr 末尾附在错误里——多为未登录/缺 Key。补鉴权（§2）。 |
| `agent did not return a sessionId` | CLI 版本与 ACP 协议不匹配，或需要 `authenticate`。核对 `hermes acp` 文档，必要时设 `HERMES_ACP_AUTH_METHOD`。 |
| 探测卡顿 | 某些 CLI 无 `--version`；探测有 6s 超时并会杀掉残留进程，安全但略慢。 |

## 7. 本环境已验证 vs 需真机验证

- **已在本仓库自动化验证**（`tests/test_real_cli_integration.py`）：用一个**忠实的 `hermes` 替身**（响应 `hermes --version` 与 `hermes acp`、说真 ACP 协议）放上 PATH，证明**生产代码路径**正确——PATH 探测 → 注册 `acp_cli`（非 mock）→ spawn `hermes acp` → initialize/session/prompt → 流式 + 产物文件。并在 `ACP_ALLOW_MOCK_FALLBACK=false` 的生产姿态下实测全链路通过。
- **需在有外网 + 凭证的真机验证**：NousResearch CLI 的真实安装与**真实模型输出**（本环境无外网、无模型凭证，无法覆盖）。按 §2–§3 在你的机器上执行即可。
